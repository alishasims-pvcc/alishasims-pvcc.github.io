All About GitHub
    Q. What is a version control system?
    A. A tool that tracks changes to files and code over time, allowing developers
    to collaborate on projects and recover earlier versions of the code.
    Q. What is Git?
    A. A free, open-source version control system (VCS) that helps developers
    track and manage changes to their code and projects
    Q.What is GitHub?
    A. A web-based platform that allows developers to store, share, and collaborate on code.
    Q. What is the difference between Git and GitHub?
    A. Git is a version control system that allows developers to track changes
    in their code. Github is a web-based hosting service for git repositories.
    Q. Who started GitHub and how was it started?
    A. It was created as a side project by Chris Wanstrath and Tom Preston-Werner
    in 2007 to improve Git.
    Q. What company owns it now?
    A. Microsoft.
    Q. How much does a GitHub account cost?
    A. Free.
    Q. What is the Octocat?
    A. The mascot for GitHub.

    Git/GitHub terms:
  • repository: the most basic element of GitHub. Contains all of the project files (including
    docs), and stores each file's revision history.
  • commit: an individual change to a file (or setof files).
  • fork: a personal copy of another user's repository that lives on your account.
  • push: to sent your committed changes to a remote resository on GitHub.com.
  • pull requests: proposed changes to a repository submitted by a user and accepted or
    rejected by a repsitory's collaborators.
  • workflows: a configurable automated process that will run one or more jobs.
  • issues: suggested improvements, tasks or questions related to the repository.
  • raw button: view or copy the raw content of a file without any styling.
  • blame button: used to examine the contents of a file line by line
    and see when each line what last modified and who the author of the modifications was.
