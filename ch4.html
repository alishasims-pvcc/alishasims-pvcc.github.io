Chapter 4: Networking

Alisha Sims

Item Definition
Network Classifications
  • LAN: "local area network" - normally consists of a collection of computers in a single building or building complex
  • MAN: "metropolitan area network" - a network of intermediate size, such as one spanning a local community.
  • WAN: "wide area network" - links machines over a greater distance - perhaps in neighboring cities or on opposite sides of the world.
    		•
    Network models
  • bus network: two or more popular topologies, in which the machines are all connected to a common communication line.
  • star network: on machice serves as a central focal point to which all the others are connected.
    		•
    Protocols
  • CSMA/CD Ethernet: "carrier sense, multiple access with collision detection" - the protocol that controls the right to transmit messages.
  • CSMA/CA Wifi: "carrier sense, multiple access with collision avoidance" - wireless networks that adopt the policy of trying to avoid collisions rather that trying to detect them.
    		•
    Combining networks
  • repeater: a device that passes signals back and forth between the two originial buses (usually with some form of amplification) without considering the meaning of the signals.
  • bridge: like a repeater, it connects two buses, but it does not necessarily pass all messages across the connection.
  • switch: a bridge with multiple connections, allowing it to connect several buses rather than just two.
  • router: devices that handle the connection between networks to form an internet.
  • gateway: the "point" at which one network is linked to an internet because it serves as a passageway between the network and the outside world.
    		•
    Client/server model
  • p2p: peer-to-peer: the client/server model involves one process (the server) providing a service to numerous others (clients). Involves processes that provide service to and receive service from each other.
  • distributed system: consist of of software units that execute as processes on different computers.
  • cloud computing: huge pools of shared computers on the network can be allocated for use by clients as needed, is the latest trend in distributed systems.
    		•
    Internet Addressing
  • Tier 1: top hierarchy ISP's that consist of very high-speed, high-capacity, international WAN's.
  • Tier 2: connecting to the tier-1 ISP's, tend to be more regional in scope and less potent in their capabilities.
  • Tier 3: an independent internet, sometime called an intranet, operated by a single authority that is in the business of supplying internet access to individual homes and businesses.
    		•
    IP address
  • ICANN "internet corporation for assigned names and numbers" - a nonprofit corporation established to coordinate the internet's operation.
  • DNS: "domain name system" - name servers used as an internet-wide directory system.
  • TLD: "top-level domain" - domain name classifications
  • Country-code TLDs> two letter, country name domain classifications.
    		•
    Internet protocols
  • FTP: "file transfer protocol" - an application for accessing another computer from a great distanceusing the telnet protocol
  • HTTP: "hyper text tranfer protocol" - power that webpages use to handle traditional network applications.
  • SMTP: "simple mail transfer protocol" - a way that two computers on the network may interact when transmitting an email message from one host to the other.
    		•
    World wide web
  • URL: "uniform resource locator" - a unique address give to documents to be located and retrieved on the web.
  • client-side activities: activities performed by a client (such as a browser).
  • server-side activities: activities performed by a server (such as a webserver).
    		•
    Internet four-layer software
  • application layer: where a message typically originates (or received).
  • transport layer: used by the application layer to send and receive messages over the internet.
  • network layer: accepts messages from the application layer and ensures the messages are properly formatted for transmission over the internet.
  • link layer: the responsibility of transferring the packet. Must deal with the communication details particular to the individual network in which the computer resides.
    		•
    Internet is
  • packet-switching network: the internet, through which digital data is sent by breaking the data into blocks of bits, which contain both the data being transmitted and control information for routing the data.
  • fault tolerant: the cumulative result of these features for transporting, routing, and fowarding packets is an internet that is tolerant of many types of faults, and that can fall back to other redundant pathways for delivery when congestion or failure occur.
  • redundant: most or all of the network remains connected even when very important tier-1 routers fail.
    		•
    Cybersecurity
  • malware: malicious software.
  • virus: software that infects a computer by inserting itself into programs that already reside in the machine.
  • worm: an autonomous program that transfer itself through a network, taking up residence in computers and forwarding copies of itself to other computers.
  • Trojan horse: program that enters a computer system disguised as a desireable program, such as a game or useful utility package, that is willingly imported by the victim.
  • spyware: another form of malicious software (sometimes called sniffing software), which is software that collects information about activities at the computer on which it resides and reports that information back to the instigator of the attack.
  • phishing: a technique of obtaining information explicitly by simply asking for it.
  • DDoS attack: "distributed denial of service" attack - when the work of generating the attacks is distributed over many hosts.
    		•
    Protection
  • firewall: a program or primary prevention technique used to filter traffic passing through a point in the network.
  • spam filter: firewalls designed to block unwanted email.
  • proxy server: a software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
  • antivirus software: used to detect and remover the presence of known viruses and other infections.
  • cryptography: the study of sending and receiving messages securely in the presence of adversaries.
  • public key encryption: techniquese by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
